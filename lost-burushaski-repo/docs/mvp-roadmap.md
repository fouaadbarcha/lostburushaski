# MVP Roadmap

## Phase 1: Core Lexicon and Q&A
Build:
- dialects
- concepts
- words
- variants
- basic search
- ask a word
- answers

Goal:
Get a working dialect-aware dictionary and question system live.

## Phase 2: Quality and Community Controls
Build:
- ratings
- endorsements
- duplicate detection
- notifications
- moderation tools

Goal:
Improve answer quality and reduce noisy or duplicate contributions.

## Phase 3: Search Intelligence and Standardization
Build:
- improved semantic matching
- stronger ranking
- standard candidate pipeline
- review and promotion system

Goal:
Start identifying trustworthy candidates for Standard Burushaski.

## Phase 4: Learning Layer
Build:
- courses
- lessons
- quizzes
- learner dashboard
- progress tracking

Goal:
Turn validated entries into a real learning platform.

## Suggested Early Priorities
1. Keep dialect data clean
2. Build concept linking correctly
3. Avoid a weak search foundation
4. Add duplicate prevention before question volume grows
5. Do not automate Standard Burushaski too early
